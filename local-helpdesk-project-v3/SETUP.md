# Настройка общего хранилища (JSONBin.io)

## Зачем это нужно

Без настройки каждый пользователь видит **свой локальный** склад и заявки —
данные хранятся в `localStorage` браузера и не передаются между устройствами.

После настройки JSONBin все пользователи работают с **единым облачным хранилищем**:
замена картриджа у одного сотрудника сразу отображается у всех остальных.

---

## Шаг 1 — Регистрация

1. Откройте **https://jsonbin.io**
2. Нажмите **Sign Up** → зарегистрируйтесь (бесплатно)
3. Подтвердите email

---

## Шаг 2 — Получение API Key

1. Войдите в аккаунт
2. Нажмите на иконку профиля → **API Keys**
3. Скопируйте **Master Key** (начинается с `$2b$...`)

---

## Шаг 3 — Создание хранилища (Bin)

1. Нажмите **+ Create Bin**
2. Вставьте в поле начальный JSON:

```json
{
  "tickets": [],
  "cartstock": {},
  "replacements": [],
  "overrides": {},
  "stockSeeded": false
}
```

3. Нажмите **Create Bin**
4. В адресной строке появится ID вида: `6650abc123def456789` — скопируйте его

---

## Шаг 4 — Прописать в проект

Откройте файл `js/storage.js`, найдите блок CONFIG в начале файла:

```javascript
const CONFIG = {
  BIN_ID:  '',   // ← вставьте сюда ID вашего bin
  API_KEY: '',   // ← вставьте сюда Master Key
  SYNC_INTERVAL: 30000,
};
```

Заполните:

```javascript
const CONFIG = {
  BIN_ID:  '6650abc123def456789',
  API_KEY: '$2b$10$ваш_ключ_здесь',
  SYNC_INTERVAL: 30000,
};
```

---

## Шаг 5 — Разместить файлы

Проект нужно разместить на общедоступном сервере или расшарить файлы.

### Вариант A — SprintHost (у вас уже есть)

1. Подключитесь по FTP к вашему хостингу
2. Загрузите папку `local-helpdesk-project-v3/` в нужный каталог
3. Откройте `http://ваш-домен/local-helpdesk-project-v3/`

### Вариант B — Локальный веб-сервер (для тестирования)

В папке проекта выполните:

```bash
python3 -m http.server 8080
```

Затем откройте: `http://localhost:8080`

**Важно:** при открытии через `file://` JSONBin работает, но могут быть ограничения CORS.

---

## Как работает синхронизация

| Событие | Действие |
|---|---|
| Открытие страницы | Загрузка актуальных данных из JSONBin |
| Создание заявки | Сохранение локально + отправка в JSONBin |
| Замена картриджа | Списание с общего склада + отправка в JSONBin |
| Каждые 30 секунд | Автоматическое обновление данных из JSONBin |
| Кнопка ⟳ в Helpdesk | Принудительное обновление |

---

## Лимиты бесплатного плана JSONBin

- **10 000 запросов/месяц** — достаточно для небольшой компании
- **1 Bin** бесплатно
- Данные хранятся без ограничений по времени

При превышении лимита запросы будут отклоняться — в этом случае перейдите на платный план
или разверните простой PHP/SQLite бэкенд на SprintHost.

---

## Индикатор синхронизации

В Helpdesk в шапке отображается статус:

- 🟡 **Синхронизация…** — идёт обмен данными
- 🟢 **Синхр. 14:30** — последняя успешная синхронизация
- 🔴 **Ошибка синхр.** — нет соединения или неверный ключ
- ⚠️ **Не настроено** — CONFIG не заполнен, работает только локально
